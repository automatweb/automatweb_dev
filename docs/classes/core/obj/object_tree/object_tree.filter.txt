filters the object tree, or creates a new tree from the filter

<b>errors:</b>
- if creating a new tree, no parent parameter is passed, error is thrown

<b>parameters:</b>
- array of filter parameters, according to which to filter the object tree, required
- whether to filter the current tree (true) or create a new one (false), optional,
defaults to creating a new one
type: boolean


<b>returns:</b>
none

<b>examples:</b>
#php#

$ot = new object_tree(array(
"parent" => 5,
"name" => "%foo%",
"class_id" => CL_BAR,
"cool_property" => true
));

$ot->filter(array(
"status" => STAT_ACTIVE
)), false); // filters the current tree for all objects that are active

$ot->filter(array(
"parent" => 90,
"status" => STAT_ACTIVE
)), true); // creates a new tree of all the objects below folder 90 that are active

#/php#
